# 設問：その他ネットワーク設計⑧（出典：設計書_md/07_管理.md）

> 出典：①ネットワーク要件・詳細設計書(TC).pptx スライド48

## 該当する要件
すべてのデバイスに以下の管理機能を設定する
- SNMP（コミュニティ、Trap）
  - コミュニティ名：TSHOOT
  - 権限：RW
  - Trapを送信する宛先：10.1.100.1
  - Loopbackインターフェイスを持つデバイスは送信元アドレスをLoopbackのアドレスに設定

## 【設問】
> 設定するコマンドを示してください。

## 回答

### 全デバイス共通

```
Device(config)# snmp-server community TSHOOT RW
Device(config)# snmp-server enable traps
Device(config)# snmp-server host 10.1.100.1 version 2c TSHOOT
```

- `snmp-server community TSHOOT RW` … コミュニティ名TSHOOT、権限RW（Read-Write）
- `snmp-server host 10.1.100.1 version 2c TSHOOT` … Trap送信先10.1.100.1へ、同じコミュニティ名でTrapを送る
- `snmp-server enable traps` … 各種Trap通知を有効化（種類を絞る場合は `snmp-server enable traps <keyword>` の形で個別指定も可）

### Loopbackインターフェイスを持つデバイス（R1, R2, CSW1, CSW2, DSW1, DSW2）のみ追加

```
Device(config)# snmp-server trap-source Loopback0
```

Loopbackを持たないASW1・ASW2・SVSWにはこのコマンドは設定不要（送信元は実際に発信するインターフェースのアドレスがそのまま使われる）。

### パラメータまとめ
| パラメータ | 値 |
|---|---|
| コミュニティ名 | TSHOOT |
| 権限 | RW |
| Trap送信先 | 10.1.100.1 |
| Trap送信元（Loopback保有機器のみ） | Loopback0 |

### 補足
- `snmp-server trap-source` はTrap（デバイス→管理サーバー方向の通知）の送信元インターフェースを固定するコマンドであり、監視サーバー側でLoopbackアドレス（10.1.254.x）を送信元として認識できるようにする。ループバックが物理リンク障害の影響を受けない安定したアドレスであるため、これを送信元にするのが定石。
- 本番環境ではコミュニティ文字列によるSNMPv1/v2cは平文でセキュリティが弱いため、可能であればSNMPv3への移行が推奨されるが、本要件はv2c相当（コミュニティ名指定）を前提とする。
