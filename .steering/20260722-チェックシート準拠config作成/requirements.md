# 要求内容

## 概要

`設計書_md/` のネットワーク設計を実際に構築できるよう、`④チェックシート.xlsx` のカテゴリ②〜⑦に対応した Cisco コンフィグ一式を `config/` 配下に作成する。カテゴリごとにフォルダを分け、その中に機器ごとの `.txt` コンフィグと、カテゴリ単位の確認コマンド（show 系）を用意する。さらに機器ごとに全カテゴリを結合した統合 running-config も作成する。

## 背景

- 設計書（物理設計・論理設計・耐障害性・可用性・セキュリティ・管理・拠点間接続設計）は体系化済みだが、実際に機器へ投入するコンフィグ成果物がまだ `config/` に存在しない。
- `④チェックシート.xlsx` は「カテゴリ（②〜⑦）× 機器（ASW1〜監視サーバ）」のマトリクス構成になっており、構築・検証をカテゴリ単位で段階的に進める運用に適している。
- コンフィグをチェックシートのカテゴリ構造に合わせて分割しておくことで、「L2 を設定 → 確認 → L3 を設定 → 確認 …」という段階的な構築・トラブルシュートがしやすくなる。

## 今回の作業内容

### 1. config/ のカテゴリ別ディレクトリ構築
- `④チェックシート.xlsx` のシート名に合わせて、②〜⑦の6フォルダを `config/` 配下に作成する。
- 完了すると、カテゴリ単位でコンフィグ成果物を管理できる土台ができる。

### 2. カテゴリ別・機器別コンフィグ（.txt）の作成
- 各カテゴリフォルダに、そのカテゴリで設定が必要な機器の `.txt` を作成する（関連機器のみ）。
- 各 `.txt` には、そのカテゴリ・その機器で投入すべき Cisco コンフィグ（設定コマンド）を記載する。
- 完了すると、機器ごと・レイヤごとに投入すべきコマンドが明確になる。

### 3. カテゴリ別 確認コマンド（show系）の作成
- 各カテゴリフォルダに `確認コマンド.txt` を1枚作成する。
- 設定が正しく入ったか検証するための show コマンドと確認ポイントを、機器ごとにセクション分けして記載する。
- 完了すると、チェックシートの各カテゴリを検証する手順が揃う。

### 4. 機器ごとの統合 running-config の作成
- `config/_統合/` に、機器ごとに②〜⑦の全カテゴリ設定を結合した完全な running-config（`.txt`）を作成する。
- 完了すると、実機・シミュレータへ一括投入できる形の最終コンフィグが揃う。

## 受け入れ条件

### カテゴリ別ディレクトリ・コンフィグ
- [ ] `config/②初期設定・L2項目/` 〜 `config/⑦EGPルーティング/` の6フォルダが存在する
- [ ] 各フォルダに、対象機器分の `.txt` が過不足なく存在する（下記「対象機器の割り当て」に一致）
- [ ] 各 `.txt` の内容が `設計書_md/` および対応する `設問/` の記載と矛盾しない

### 確認コマンド
- [ ] 各カテゴリフォルダに `確認コマンド.txt` が1枚ずつ存在する
- [ ] 各 `確認コマンド.txt` に、そのカテゴリで設定した内容を検証できる show コマンドが機器ごとに記載されている

### 統合config
- [ ] `config/_統合/` に、コンフィグ対象となる各機器の統合 running-config が存在する
- [ ] 統合configの内容が、カテゴリ別configを結合したものと一致する

## 成功指標

- チェックシート②〜⑦の各カテゴリについて、「投入コンフィグ」と「確認手順」がセットで揃っている
- 設計書のIPアドレス表・OSPFエリア設計・HSRP/BGPパラメータ等と、生成したコンフィグの値が完全に一致している
- 統合configをそのまま機器へ投入すれば設計どおりのネットワークが構成できる状態になっている

## 対象機器の割り当て（カテゴリ別）

| フォルダ | 対象機器 |
|---|---|
| ②初期設定・L2項目 | ASW1, ASW2, DSW1, DSW2, SVSW |
| ③L３項目 | DSW1, DSW2, CSW1, CSW2, R1, R2 |
| ④IGPルーティング | DSW1, DSW2, CSW1, CSW2, R1, R2 |
| ⑤セキュリティ | ASW1, ASW2, DSW1, DSW2 |
| ⑥管理項目 | ASW1, ASW2, DSW1, DSW2, CSW1, CSW2, R1, R2, SVSW |
| ⑦EGPルーティング | DSW1, DSW2, CSW1, CSW2, R1, R2 |

※ PC1・PC2（DHCPクライアント）、監視サーバ（NTP/Syslog/SNMPサーバ）はエンドホストのためコンフィグ対象外。

## スコープ外

以下は今回の作業では扱いません:

- `④チェックシート.xlsx` 本体への確認項目の記入（今回は `config/` 配下のファイル群のみを作成。xlsxは雛形のまま残す）
- ①物理配線（物理結線作業でありコンフィグ対象外）
- PC・監視サーバ側の設定（エンドホスト）
- オプション設定（`設計書_md/09_オプション.md`）の実装 ※必要なら別作業で追加
- 実機・シミュレータでの実投入と動作確認（コンフィグ成果物の作成まで）
