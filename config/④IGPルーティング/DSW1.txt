! ============================================================
! ④ IGPルーティング : DSW1 （ディストリビューションL3SW / ABR）
! 出典: 設計書_md/03_論理設計.md（論理設計⑧⑨）
!       設問 02_パッシブインターフェース / 03_OSPFネットワークタイプ最適化
! 内容: OSPF（エリア0/1境界, MD5認証, router-id手動, passive, P2P）
! 前提: L3 IP/SVI/Po は ③ で設定済み
! ============================================================
!
! ------------------------------------------------------------
! ■ 不明点・確認事項（実機設定時に確認）
!  - エリア設計（論理設計⑧）: DSW-CSW-R間=エリア0、DSW配下のクライアント/管理
!    セグメント=エリア1。DSWはABR。ループバックはエリア0に収容。
!  - Po3(DSW間)はL2トランクでL3 IFを持たないためOSPF隣接は張らない。設問02の
!    例は Po3 を no passive に挙げるが、本設計ではPo3はL2のため対象外。DSW1-DSW2
!    間の経路交換はCSW経由(エリア0)で成立する（直接隣接は不要）。
!  - MD5認証: area 0 authentication message-digest + 各エリア0リンクに
!    message-digest-key 1 md5 cisco123（パスワードは設計指定 cisco123）。
!  - passive-interface default で全IFをpassive化し、CSW向けアップリンク
!    (Po1/Po2)のみ解除（設問02準拠）。クライアントVLANはpassiveのまま経路広告。
!  - /30リンクは ip ospf network point-to-point でDR/BDR排除（設問03準拠）。
! ------------------------------------------------------------

hostname DSW1
!
! ============================================================
! エリア0リンクのOSPF設定（P2P + MD5）
!   Po1: DSW1-CSW1(10.1.255.0/30) / Po2: DSW1-CSW2(10.1.255.16/30)
! ============================================================
interface Port-channel1
 ip ospf network point-to-point
 ip ospf message-digest-key 1 md5 cisco123
!
interface Port-channel2
 ip ospf network point-to-point
 ip ospf message-digest-key 1 md5 cisco123
!
! ============================================================
! OSPFプロセス（router-id 1.1.1.1 / 論理設計⑧）
! ============================================================
router ospf 1
 router-id 1.1.1.1
 area 0 authentication message-digest
 passive-interface default
 no passive-interface Port-channel1
 no passive-interface Port-channel2
 network 10.1.254.1 0.0.0.0 area 0
 network 10.1.255.0 0.0.0.3 area 0
 network 10.1.255.16 0.0.0.3 area 0
 network 10.1.10.0 0.0.0.255 area 1
 network 10.1.20.0 0.0.0.255 area 1
 network 10.1.30.0 0.0.0.255 area 1
 network 10.1.40.0 0.0.0.255 area 1
 network 10.1.50.0 0.0.0.255 area 1
 network 10.1.60.0 0.0.0.255 area 1
 network 10.1.99.0 0.0.0.7 area 1
!
end
