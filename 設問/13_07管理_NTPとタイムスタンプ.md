# 設問：その他ネットワーク設計⑦（出典：設計書_md/07_管理.md）

> 出典：①ネットワーク要件・詳細設計書(TC).pptx スライド47

## 該当する要件
すべてのデバイスに以下の管理機能を設定する
- NTPサーバーの指定（NTPサーバーのアドレス：10.1.100.1）
- タイムスタンプ（logメッセージとdebugメッセージで表示される時間を現在の時刻に設定）

## 【設問】
> 設定するコマンドを示してください。

## 回答

対象は全機器（R1, R2, CSW1, CSW2, DSW1, DSW2, ASW1, ASW2, SVSW）。

### NTPサーバーの指定

```
Device(config)# ntp server 10.1.100.1
```

### タイムスタンプ（log／debugメッセージに現在時刻を付与）

```
Device(config)# service timestamps log datetime msec localtime
Device(config)# service timestamps debug datetime msec localtime
```

- `datetime` … 相対的な経過時間（uptime）ではなく実時刻を表示
- `msec` … ミリ秒まで表示（障害解析時の時系列の精度を上げる）
- `localtime` … NTPで同期したUTC時刻をローカルタイムゾーンで表示（`clock timezone` 未設定の場合はUTCのまま表示される点に注意）

### 補足
- `service timestamps` はデフォルトで `uptime`（起動からの経過時間）表示になっているため、明示的に `datetime` へ変更しないと、NTPで時刻同期しても実時刻としてログに残らない。
- タイムゾーンを日本時間として明示したい場合は `clock timezone JST 9` を追加するとよい（要件には明記されていないため任意）。
- `show ntp status` / `show clock detail` でNTP同期状態（synchronized）と現在時刻設定を確認できる。
