# 設問：その他ネットワーク設計① ①（出典：設計書_md/04_耐障害性.md）

> 出典：①ネットワーク要件・詳細設計書(TC).pptx スライド38

## 該当する要件
- DSWはVLAN間ルーティングを行う重要な役割を持つため、二重化して耐障害性を高める
- **ASW－DSW間のL2の障害対策として、障害時の通信断を最小限に抑えるための機能を導入し、VLANごとにトラフィックが負荷分散されるようにする**

## 【設問】
> どのような技術を使用しますか？　その技術に関わるパラメータは、具体的にどのような値にしますか？（コマンドでも説明でも可）

## 回答

### 使用する技術：Rapid PVST+（IEEE 802.1w、VLANごとのSTPインスタンス）＋ VLANごとのルートブリッジ分散

物理構成図の通り、ASW1/ASW2はDSW1・DSW2の両方に接続されており（Gi1/0/5,6のクロス配線）、L2ループが発生する構成になっている。ループ防止にはSTPが必須だが、単純に1つのツリー（CST）でDSW1のみをルートにすると、DSW2への物理リンクは常にブロッキングとなり、帯域を活用できず「VLANごとの負荷分散」の要件を満たせない。

そこで **PVST+（VLANごとに独立したSTPインスタンス）** を用い、VLANグループごとにルートブリッジを使い分ける。

- **DSW1** … VLAN 10, 20, 30, 99 のルートブリッジ（プライマリ）
- **DSW2** … VLAN 40, 50, 60 のルートブリッジ（プライマリ）
- 互いに他方のVLANグループのセカンダリルートブリッジになる

これにより、VLAN10/20/30/99のトラフィックはDSW1側のアップリンクを主として使用し、VLAN40/50/60のトラフィックはDSW2側のアップリンクを主として使用するため、物理的な2本のリンク（DSW1向け・DSW2向け）の両方が実際に使われる状態になり、負荷分散が実現する。

また「障害時の通信断を最小限に抑える」ため、従来のPVST（802.1D、収束に30〜50秒）ではなく、高速収束（1〜3秒程度）が可能な **Rapid PVST+（802.1w）** を使用する。

### 設定コマンド例

```
! 全SW共通：モードをRapid PVST+へ
Switch(config)# spanning-tree mode rapid-pvst

! DSW1：VLAN10,20,30,99のルートプライマリ／VLAN40,50,60はセカンダリ
DSW1(config)# spanning-tree vlan 10,20,30,99 root primary
DSW1(config)# spanning-tree vlan 40,50,60 root secondary

! DSW2：VLAN40,50,60のルートプライマリ／VLAN10,20,30,99はセカンダリ
DSW2(config)# spanning-tree vlan 40,50,60 root primary
DSW2(config)# spanning-tree vlan 10,20,30,99 root secondary
```

`root primary` はブリッジプライオリティを24576（既存最小値より小さければ自動調整）、`root secondary` は28672に設定するマクロコマンド。直接 `spanning-tree vlan X priority Y` で明示的な値（例：4096 / 8192）を指定してもよい。

### 補足
- ルートブリッジの配置は、物理構成図に記載されている「root（VLAN10,20,30,99）＝DSW1」「root（VLAN40,50,60）＝DSW2」という注記と整合させる必要がある。
- ルートブリッジの配置は、後述のHSRP（PC-DSW間のデフォルトゲートウェイ冗長化）のActive側とも一致させるのが望ましい（STPルート＝L2最短経路の起点と、HSRP Active＝L3ゲートウェイを同一機器にすることで、不要な機器間トラフィック＝トロンボーン現象を避けられる）。
