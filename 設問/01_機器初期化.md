## 【設問】初期化されているかどうかを、どのように確認しますか？

コンソールケーブルでPCと接続し、ターミナルソフト（Tera Term等）から確認する。工場出荷状態（初期化済み）の機器には、以下のような特徴が現れる。

### 1. 起動時のダイアログで判断する
電源投入直後、コンフィグが無い（NVRAMが空）機種では自動的に初期設定ダイアログが表示される。

```
Would you like to enter the initial configuration dialog? [yes/no]:
```

ここが表示される、あるいは表示をスキップしても特権モードのプロンプトが**デフォルトのホスト名**（`Router>` や `Switch>`）になっている場合は初期化済みと判断できる。逆にホスト名が `R1>` `CSW1>` のように既に設定されていれば、初期化されていない（前のチームの設定が残っている等）。

### 2. コマンドで確認する
特権モードに入り（後述の通りパスワード無しで入れる前提）、以下を確認する。

```
Router> enable
Router# show startup-config
```

- `startup-config is not present` と表示される → NVRAM上に保存済み設定が無い＝初期化済み
- 何らかの設定内容が表示される → 初期化されていない

```
Router# show running-config
```
- ホスト名がデフォルト（Router/Switch）、`enable secret`/`enable password` が無い、インターフェースにIPアドレスが設定されていない、等であれば初期化済みと判断できる。

**スイッチ特有の確認項目：**

```
Switch# show vlan brief
```
- VLAN1（default）以外のVLANが登録されていなければ初期化済み。VLAN10、VLAN99など演習で使うVLANが既に存在していれば、`vlan.dat`（VLANデータベース）が初期化されていない。

```
Switch# show vtp status
```
- Configuration Revision（VTPリビジョン番号）が `0` であれば、VLANデータベースがクリアな状態。

### まとめ
| 確認コマンド | 初期化済みの状態 |
|---|---|
| `show startup-config` | `startup-config is not present` |
| `show running-config` | ホスト名がデフォルト、パスワード類が未設定 |
| `show vlan brief`（SWのみ） | VLAN1のみ存在 |
| `show vtp status`（SWのみ） | リビジョン番号 0 |

## 【設問】パスワードは問題なく特権モードへアクセスできる前提で、機器を初期化する方法やコマンドを説明してください。

特権モード（`#`）にアクセスできる前提で、**startup-config（NVRAM）とVLANデータベース（flash:vlan.dat）を削除し、再起動する**ことで工場出荷状態に戻す。

### ルータの初期化

```
Router> enable
Router# erase startup-config
```
実行すると以下の確認が出るので `Enter` でそのまま進める。
```
Erasing the nvram filesystem will remove all configuration files! Continue? [confirm]
```

続いて再起動する。
```
Router# reload
```
`System configuration has been modified. Save?` と聞かれた場合は **`no`** を選択する（ここで `yes` にすると running-config が再度 startup-config に保存されてしまい、初期化が無効になるため注意）。
```
Proceed with reload? [confirm]
```
再起動後、初期設定ダイアログ（`Would you like to enter the initial configuration dialog?`）が出れば初期化完了。

### L2SW / L3SW の初期化（2960 / 3560 / 3750）

スイッチは **VLAN情報が startup-config ではなく `flash:vlan.dat` に別管理されている**ため、`erase startup-config` だけでは不十分。`vlan.dat` も削除する必要がある。

```
Switch> enable
Switch# erase startup-config
Switch# delete flash:vlan.dat
```
削除確認が出るので `Enter` で進める。
```
Delete filename [vlan.dat]?
Delete flash:vlan.dat? [confirm]
```

最後に再起動する。
```
Switch# reload
```
同様に「Save?」と聞かれたら **`no`**。

### コマンドまとめ

| 機器 | 実行コマンド |
|---|---|
| ルータ（R1/R2） | `erase startup-config` → `reload`（保存は No） |
| L2SW/L3SW（ASW/DSW/CSW/SVSW） | `erase startup-config` → `delete flash:vlan.dat` → `reload`（保存は No） |

### 補足（なぜ vlan.dat の削除が必要か）
Catalystスイッチでは、VTP（VLAN Trunking Protocol）が管理するVLANデータベースは `running-config`/`startup-config` とは独立したファイル `flash:vlan.dat` に保存される仕組みになっている。そのため `erase startup-config` のみを実行して再起動しても、以前作成したVLAN（VLAN10、VLAN99など）が復元されてしまう。演習を正しい初期状態から始めるためには、必ず `vlan.dat` も削除すること。
