# 設問：その他ネットワーク設計⑤ ①（出典：設計書_md/06_セキュリティ.md）

> 出典：①ネットワーク要件・詳細設計書(TC).pptx スライド44

## 該当する要件
- **ASWでは不正なDHCPサーバーが接続されるのを防ぐ仕様とする**

## 【設問】
> どのような技術を使用しますか？　その技術に関わるパラメータは、具体的にどのような値にしますか？（コマンドでも説明でも可）

## 回答

### 使用する技術：DHCP Snooping

クライアントは自動構成（DHCP）でIPアドレスを取得する設計であり（論理設計②）、DSWが各VLANのDHCPサーバー役を担う。もしクライアントポートに不正なDHCPサーバーが接続されると、クライアントに誤ったIP・デフォルトゲートウェイ・DNSを配布し、通信傍受や通信断を引き起こす恐れがある。**DHCP Snooping** を有効にすると、指定した信頼ポート（trust）以外から届くDHCP OFFER/ACK等のサーバー応答メッセージを破棄できる。

### 設定コマンド例（ASW1の場合）

```
ASW1(config)# ip dhcp snooping
ASW1(config)# ip dhcp snooping vlan 10,20,30,40,50,60,99

! DSW向けアップリンクのみ信頼ポートとする
ASW1(config)# interface range GigabitEthernet1/0/5 - 6
ASW1(config-if-range)# ip dhcp snooping trust

! クライアント接続ポート（デフォルトでuntrust）は明示不要だが、必要なら
ASW1(config)# interface range FastEthernet0/1 - 6
ASW1(config-if-range)# no ip dhcp snooping trust
```

DSW側もDHCPサーバー機能を持つため、DSW1/DSW2でも `ip dhcp snooping` を有効化し、対象VLANを指定する。

### パラメータのポイント
| パラメータ | 値 |
|---|---|
| 有効化対象VLAN | 10,20,30,40,50,60,99（クライアント／管理VLAN全て） |
| trustポート | ASWからDSWへのアップリンク（正規のDHCPサーバー応答が通る経路） |
| untrustポート（デフォルト） | クライアントPC接続ポート（FE0/1〜FE0/6） |

### 補足
- DHCP Snooping有効化後、正規のDHCPサーバー（DSW）からの応答がtrustポート経由で届くことを確認すること。trust設定を誤るとクライアントが正規のDHCPサーバーからもアドレスを取得できなくなる。
- DHCP Snoopingを有効にすると自動的にDHCPスヌーピングバインディングテーブルが作成され、後述のポートセキュリティやIP Source Guard等と連携できる。
