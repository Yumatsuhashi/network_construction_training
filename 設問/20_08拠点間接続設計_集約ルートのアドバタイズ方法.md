# 設問：拠点間接続設計③ ②（出典：設計書_md/08_拠点間接続設計.md）

> 出典：①ネットワーク要件・詳細設計書(TC).pptx スライド53

## 該当する要件
- IP-VPN網へは**集約ルート10.1.0.0/16をアドバタイズする**
- （前提）DSWやCSWのBGPテーブルに192.168のルートや不要な詳細ルートが存在しないこと

## 【設問】
> 集約ルートをアドバタイズする方法を2種類、コマンドとともに示してください。

## 回答

R1・R2それぞれで、内部の詳細経路（10.1.10.0/24, 10.1.20.0/24…等）をそのまま流すのではなく、**10.1.0.0/16という1つの集約ルート**としてIP-VPN網（eBGPネイバー）へ広告する。方法は大きく2種類ある。

### 方法1：静的経路＋networkコマンド

BGPの `network` コマンドは、ルーティングテーブル（RIB）に**完全一致**するエントリが存在しないと経路を広告できない仕様のため、まずNull0への集約静的経路を作成し、それをBGPに広告させる。

```
R1(config)# ip route 10.1.0.0 255.255.0.0 Null0
R1(config)# router bgp 65001
R1(config-router)# network 10.1.0.0 mask 255.255.0.0
```

R2にも同様に設定する（両ルータから同じ集約ルートを広告することで、IP-VPN網側から見ても冗長経路になる）。

### 方法2：aggregate-addressコマンド

BGPプロセス内で直接、詳細経路群から集約ルートを自動生成させる方法。

```
R1(config)# router bgp 65001
R1(config-router)# aggregate-address 10.1.0.0 255.255.0.0 summary-only
```

- `summary-only` を付けることで、集約後の**詳細経路（10.1.10.0/24等の個別ルート）をeBGPネイバーへ広告しない**ようにできる。これを付けない場合、集約ルートに加えて詳細ルートも一緒に広告されてしまい、「集約ルートのみを流す」という要件に反する。

### 2つの方法の違い
| | 方法1（network＋静的経路） | 方法2（aggregate-address） |
|---|---|---|
| 前提条件 | RIBに完全一致するエントリ（Null0への静的経路）が必要 | BGPテーブルに集約対象となる詳細経路（サブネット）が存在すればよい |
| 詳細経路の抑制 | 元々詳細経路をBGPに個別広告していなければ自然と集約ルートのみになる | `summary-only` オプションで明示的に抑制する必要がある |
| 設定箇所 | グローバルコンフィグ（`ip route`）＋BGPコンフィグ | BGPコンフィグのみで完結 |

### 補足
- どちらの方法でも、内部のOSPFで学習した詳細経路（10.1.10.0/24等）をBGPへ`redistribute`しないことが前提。redistributeしてしまうと、`summary-only`を付けていても内部からの詳細情報がBGPプロセスに流入し、意図せず経路が漏れる設計ミスにつながりやすいため注意する。
