# 設問：その他ネットワーク設計⑤ ②（出典：設計書_md/06_セキュリティ.md）

> 出典：①ネットワーク要件・詳細設計書(TC).pptx スライド44

## 該当する要件
- **クライアントPCが接続されているポートは不正なホストが接続されるのを防ぐ仕様とする**

## 【設問】
> どのような技術を使用しますか？　その技術に関わるパラメータは、具体的にどのような値にしますか？（コマンドでも説明でも可）

## 回答

### 使用する技術：Port Security

ASWのクライアント接続ポート（FE0/1〜FE0/6）に、想定していないホスト（不正な機器・PC）が接続されることを防ぐため、**ポートセキュリティ** を用いてポートごとに許可するMACアドレス数を制限する。

### 設定コマンド例（ASW1の場合）

```
ASW1(config)# interface range FastEthernet0/1 - 6
ASW1(config-if-range)# switchport mode access
ASW1(config-if-range)# switchport port-security
ASW1(config-if-range)# switchport port-security maximum 1
ASW1(config-if-range)# switchport port-security mac-address sticky
ASW1(config-if-range)# switchport port-security violation restrict
```

### パラメータのポイント
| パラメータ | 値 | 理由 |
|---|---|---|
| `maximum` | 1 | 1ポートにつきクライアントPC1台のみ接続想定のため |
| `mac-address` | sticky（動的学習して自動的にrunning-configへ記録） | 事前にMACアドレスを台帳化する手間を省き、最初に学習したMACのみ許可する |
| `violation` | restrict（または`shutdown`） | 制限超過時にログを残しつつフレームを破棄（`shutdown`ならポート自体をerr-disableにしてより厳格に遮断する選択肢もある） |

### 補足
- `violation shutdown`（デフォルト）を選んだ場合、違反発生時にポートがerr-disable状態になり、復旧には `shutdown` / `no shutdown` による手動（または `errdisable recovery` による自動）復旧が必要になる。演習・運用のどちらを優先するかで `restrict` と `shutdown` を使い分けるとよい。
- 前設問のDHCP Snoopingと組み合わせる場合、`ip dhcp snooping` のバインディング情報を使った **IP Source Guard**（`ip verify source`）を追加すると、MACだけでなくIPアドレスの詐称（なりすまし）も防止できる（本要件には明記されていないため参考情報）。
