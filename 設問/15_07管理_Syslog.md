# 設問：その他ネットワーク設計⑨ ①（出典：設計書_md/07_管理.md）

> 出典：①ネットワーク要件・詳細設計書(TC).pptx スライド49

## 該当する要件
すべてのデバイスに以下の管理機能を設定する
- Syslog
  - Syslogサーバーのアドレス：10.1.100.1
  - Loopbackインターフェイスを持つデバイスは送信元アドレスをLoopbackのアドレスに設定

## 【設問】
> 設定するコマンドを示してください。

## 回答

### 全デバイス共通

```
Device(config)# logging host 10.1.100.1
```

（IOSバージョンによっては `logging 10.1.100.1` の表記でも同義）

### Loopbackインターフェイスを持つデバイス（R1, R2, CSW1, CSW2, DSW1, DSW2）のみ追加

```
Device(config)# logging source-interface Loopback0
```

ASW1・ASW2・SVSWはLoopbackを持たないため、このコマンドは不要（デフォルトの発信インターフェースのアドレスがそのままログの送信元になる）。

### パラメータまとめ
| パラメータ | 値 |
|---|---|
| Syslogサーバーアドレス | 10.1.100.1 |
| ログ送信元（Loopback保有機器のみ） | Loopback0 |

### 補足
- 前設問（NTP／タイムスタンプ）と組み合わせることで、Syslogサーバー側で受信したログに「正確な実時刻」と「機器を一意に識別できる安定した送信元アドレス（Loopback）」の両方が付与され、障害解析時にログの突合せがしやすくなる。
- 必要に応じて `logging trap informational`（重要度レベルの指定）等も検討可能だが、本要件では明記されていないためデフォルトのまま（`debugging`＝全レベル送信）としてよい。
