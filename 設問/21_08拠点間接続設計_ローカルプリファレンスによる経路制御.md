# 設問：拠点間接続設計④ ①（出典：設計書_md/08_拠点間接続設計.md）

> 出典：①ネットワーク要件・詳細設計書(TC).pptx スライド54

## 該当する要件
- **10.2.0.0/16（拠点2）宛のトラフィックはR2を経由するように調整**し、**10.3.0.0/16（拠点3）宛のトラフィックはR1を経由するように調整**する

## 【設問】
> どのような技術を使用しますか？　その技術に関わるパラメータは、具体的にどのような値にしますか？（コマンドでも説明でも可）

## 回答

### 使用する技術：Local Preference（ローカルプリファレンス）＋ route-map

R1・R2はどちらもIP-VPN網とeBGPで接続しているため、拠点2（10.2.0.0/16）・拠点3（10.3.0.0/16）宛の経路は、R1・R2の両方がIP-VPN網から受け取る可能性がある（マルチホーム構成のため）。拠点内の全ルータ（CSW/DSWを含む）に「拠点2宛はR2経由・拠点3宛はR1経由」という一貫した経路選択をさせるには、IBGPで内部伝播する属性である **Local Preference** を使うのが適切。

（Weightではダメな理由：Weightはそのルータ自身のローカルな属性でありIBGPピアには一切伝わらないため、CSW/DSWなど他のIBGPルータの経路選択には影響を与えられない。全ノードで一貫した経路選択をさせたい本要件には、IBGPで伝播するLocal Preferenceが適している。）

### 設定コマンド例

**R2：拠点2（10.2.0.0/16）宛の経路にLocal Preferenceを高く設定**

```
R2(config)# ip prefix-list SITE2-ROUTE permit 10.2.0.0/16
!
R2(config)# route-map SET-LOCALPREF-IN permit 10
R2(config-route-map)# match ip address prefix-list SITE2-ROUTE
R2(config-route-map)# set local-preference 200
R2(config-route-map)# exit
R2(config)# route-map SET-LOCALPREF-IN permit 20
!
R2(config)# router bgp 65001
R2(config-router)# neighbor 192.168.12.1 route-map SET-LOCALPREF-IN in
```

**R1：拠点3（10.3.0.0/16）宛の経路にLocal Preferenceを高く設定**

```
R1(config)# ip prefix-list SITE3-ROUTE permit 10.3.0.0/16
!
R1(config)# route-map SET-LOCALPREF-IN permit 10
R1(config-route-map)# match ip address prefix-list SITE3-ROUTE
R1(config-route-map)# set local-preference 200
R1(config-route-map)# exit
R1(config)# route-map SET-LOCALPREF-IN permit 20
!
R1(config)# router bgp 65001
R1(config-router)# neighbor 192.168.11.1 route-map SET-LOCALPREF-IN in
```

### パラメータのポイント
| パラメータ | 値 | 説明 |
|---|---|---|
| Local Preference（デフォルト） | 100 | 明示設定していない経路はデフォルト値のまま |
| Local Preference（優先させたい経路） | 200 | デフォルトより高い値を設定し、その経路をBGPベストパスとして選ばせる |
| 適用箇所 | eBGPネイバー（IP-VPN網側）からの **inbound** route-map | 受信した経路にLocal Preferenceを付与し、以降IBGPで全ノードへ伝播させる |

### 補足
- Local Preferenceは値が**大きいほど優先**される（MEDとは逆）。
- route-mapの2行目（`permit 20` で条件なし＝全許可）を忘れると、prefix-listにマッチしない経路が全て暗黙のdenyで拒否されてしまうため必須。
