# 設問：拠点間接続設計②（出典：設計書_md/08_拠点間接続設計.md）

> 出典：①ネットワーク要件・詳細設計書(TC).pptx スライド52

## 該当する要件
- 通信サービスはIP-VPN網を使用して拠点間接続をする
- R1、R2はBGPを使用してIP-VPN網と接続する

**R2接続条件**
- IP-VPN網側のアドレス：192.168.12.2/30
- 自AS番号：65001
- ネイバーアドレス：192.168.12.1
- ネイバーAS番号：64999
- 認証パスワード：cisco123

## 【設問】
> 設定するコマンドを示してください。

## 回答

### インターフェース設定（前提）

```
R2(config)# interface GigabitEthernet0/1
R2(config-if)# ip address 192.168.12.2 255.255.255.252
R2(config-if)# no shutdown
```

### BGP設定（eBGPネイバー）

```
R2(config)# router bgp 65001
R2(config-router)# neighbor 192.168.12.1 remote-as 64999
R2(config-router)# neighbor 192.168.12.1 password cisco123
```

R1（設問①）と同一AS番号（65001）だが、IP-VPN網側のアドレス・ネイバーアドレスは別セグメント（192.168.12.0/30）になる点に注意。R1・R2ともに同じAS 65001に属しているため、拠点内では **IBGP** の関係になる（後述の拠点間接続設計③参照）。

### 補足
- R1・R2はどちらも同じ自AS番号65001でIP-VPN網（AS64999）とeBGPpiaを張る、いわゆる**マルチホーム構成**。これにより一方のルータやリンクが故障しても、もう一方の経路でIP-VPN網との接続性を維持できる（拠点間接続設計④の経路制御と合わせて冗長性・負荷分散を実現）。
