# 設問：論理設計⑨ ②（出典：設計書_md/03_論理設計.md）

> 出典：①ネットワーク要件・詳細設計書(TC).pptx スライド36

## 該当する要件
- 拠点内のルーティングプロトコルはOSPFを使用する
- **/30ネットワークはDR/BDRの選定を無くしコンバージェンスを早めるようにする**

## 【設問】
> どのような技術を使用しますか？　コマンドも示してください。

## 回答

### 使用する技術：OSPFネットワークタイプ「point-to-point」

OSPFのデフォルトのネットワークタイプは、イーサネットのようなマルチアクセス媒体では **broadcast** となり、この場合セグメント上でDR（指名ルータ）／BDR（バックアップ指名ルータ）の選定処理が発生する。DR/BDR選定は

- Helloパケットの交換とプライオリティ比較による選定シーケンス
- DR/BDR経由でのLSA中継（DRother同士は直接隣接を完全にフルにしない＝2-Way止まり）

といったオーバーヘッドを伴い、リンク障害時の再選定にも時間がかかる。

本設計の機器間リンク（DSW-CSW間、CSW-CSW間、DSW-DSW間、CSW-R間）は /30 で構成されており、リンク上に存在するルータは常に2台のみ（＝真のポイントツーポイント接続）。この場合、ネットワークタイプを **point-to-point** に変更すると、DR/BDR選定プロセスそのものが不要になり、両端が直ちにFULL状態で隣接関係を確立できるため、リンクダウン検知後の再収束（コンバージェンス）が速くなる。

### 設定コマンド例

```
DSW1(config)# interface Port-channel1
DSW1(config-if)# ip ospf network point-to-point
```

同様に、下記の全ての機器間 /30 リンクに設定する。

| 対象リンク | 設定箇所 |
|---|---|
| DSW1-CSW1（10.1.255.0/30） | DSW1 Po1 / CSW1 Fa0/1,2側 |
| DSW2-CSW2（10.1.255.4/30） | DSW2 Po1 / CSW2 Fa0/1,2側 |
| CSW1-CSW2（10.1.255.8/30） | CSW1 Po3 / CSW2 Po3 |
| CSW1-DSW2（10.1.255.12/30） | CSW1 Fa0/3,4側 / DSW2 Po2 |
| CSW2-DSW1（10.1.255.16/30） | CSW2 Fa0/3,4側 / DSW1 Po2 |
| DSW1-DSW2（該当セグメントがあれば） | DSW1 Po3 / DSW2 Po3 |
| CSW1-R1（10.1.255.20/30） | CSW1 Fa0/6 / R1 Gi0/0 |
| CSW2-R2（10.1.255.24/30） | CSW2 Fa0/6 / R2 Gi0/0 |

**重要：ネットワークタイプはリンク両端で一致させる必要がある**（片方だけpoint-to-pointにするとネイバーが形成されない）。

### 補足
- 代替案として `ip ospf priority 0` をどちらか一方に設定してDR/BDR選定から除外する方法もあるが、これは「選定を無くす」のではなく「片方を選定対象から外す」だけであり、依然としてDR/BDR自体は存在する（もう一方がDRになる）。設問の「DR/BDRの選定を無くし」という要件には、選定プロセス自体が発生しない `point-to-point` の方がより厳密に合致する。
