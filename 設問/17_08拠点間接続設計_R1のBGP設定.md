# 設問：拠点間接続設計①（出典：設計書_md/08_拠点間接続設計.md）

> 出典：①ネットワーク要件・詳細設計書(TC).pptx スライド51

## 該当する要件
- 通信サービスはIP-VPN網を使用して拠点間接続をする
- R1、R2はBGPを使用してIP-VPN網と接続する

**R1接続条件**
- IP-VPN網側のアドレス：192.168.11.2/30
- 自AS番号：65001
- ネイバーアドレス：192.168.11.1
- ネイバーAS番号：64999
- 認証パスワード：cisco123

## 【設問】
> 設定するコマンドを示してください。

## 回答

### インターフェース設定（前提）

```
R1(config)# interface GigabitEthernet0/1
R1(config-if)# ip address 192.168.11.2 255.255.255.252
R1(config-if)# no shutdown
```

### BGP設定（eBGPネイバー）

```
R1(config)# router bgp 65001
R1(config-router)# neighbor 192.168.11.1 remote-as 64999
R1(config-router)# neighbor 192.168.11.1 password cisco123
```

- `router bgp 65001` … 自AS番号65001でBGPプロセスを起動
- `neighbor 192.168.11.1 remote-as 64999` … IP-VPN網側（対向AS 64999）をネイバーとして登録。自AS（65001）と相手AS（64999）が異なるため自動的に **eBGP** として動作する
- `neighbor 192.168.11.1 password cisco123` … MD5認証（TCP MD5）でネイバー間のBGPセッションを保護

### 補足
- IP-VPN網側の対向ルータにも同じパスワード（cisco123）が設定されていなければBGPネイバー関係（Established）は確立しない。
- 内部（拠点内）へ広告する経路（集約ルート10.1.0.0/16等）やIBGPフルメッシュの設定は、後続の設問（拠点間接続設計③④）で扱う。
