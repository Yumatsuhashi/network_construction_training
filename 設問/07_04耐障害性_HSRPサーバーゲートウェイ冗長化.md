# 設問：その他ネットワーク設計③（出典：設計書_md/04_耐障害性.md）

> 出典：①ネットワーク要件・詳細設計書(TC).pptx スライド40

## 該当する要件
監視サーバーとCSW間のL3の障害対策として、独自のプロトコルを実装して、以下の条件でデフォルトゲートウェイが自動的に切り替わる仕様とする
- CSW1をActiveルータとする
- ダウンタイムを短く設定し、障害検知を早め、プライオリティが小さい状態でActiveを維持するのを防止する

## 【設問】
> どのような技術を使用しますか？　その技術に関わるパラメータは、具体的にどのような値にしますか？（コマンドでも説明でも可）

## 回答

### 使用する技術：HSRP（サーバセグメント VLAN100用）

監視サーバー・SVSWが接続されるサーバセグメント（10.1.100.0/24）のデフォルトゲートウェイを、CSW1・CSW2間でHSRP冗長化する。クライアント側（設問②）と同じ独自プロトコル（HSRP）だが、こちらは単一グループでよく、**CSW1固定でActive** とする点が異なる。

### 設定コマンド例

```
! CSW1 (Active)
CSW1(config)# interface Vlan100
CSW1(config-if)# ip address 10.1.100.253 255.255.255.0
CSW1(config-if)# standby 100 ip 10.1.100.200
CSW1(config-if)# standby 100 priority 110
CSW1(config-if)# standby 100 preempt
CSW1(config-if)# standby 100 timers msec 200 msec 750
CSW1(config-if)# standby 100 track GigabitEthernet0/0 decrement 20
```

```
! CSW2 (Standby)
CSW2(config)# interface Vlan100
CSW2(config-if)# ip address 10.1.100.254 255.255.255.0
CSW2(config-if)# standby 100 ip 10.1.100.200
CSW2(config-if)# standby 100 priority 100
CSW2(config-if)# standby 100 preempt
CSW2(config-if)# standby 100 timers msec 200 msec 750
```

### 各条件への対応

- **CSW1をActiveルータとする** → CSW1側のpriorityをCSW2より高く設定（110 > 100）
- **ダウンタイムを短く・障害検知を早める** → クライアント側と同様、`timers msec 200 msec 750` でHello/Holdをミリ秒単位に短縮
- **プライオリティが小さい状態でActiveを維持するのを防止する** → `standby preempt` を明示的に有効化（デフォルト無効のため）

### trackの考え方
CSW1がIP-VPN方向（R1）への到達性を失った場合、監視サーバーからの通信もCSW1経由では外部・拠点間通信ができなくなる。そこで、CSW1のR1向けアップリンク（例：GigabitEthernet0/0、物理構成図のFa0/6に相当）をtrackし、リンクダウン時にdecrementでpriorityを下げ、CSW2（R2経由で正常な経路を持つ）へフェイルオーバーさせる。

### 補足
- クライアント側（設問②）のような「2本のアップリンクの組み合わせ判定」の条件はこちらには明記されていないため、単純に1本のアップリンクをtrackするシンプルな構成でよい。
